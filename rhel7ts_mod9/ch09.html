9.1. Selecting a Boot Target Lab

In this lab, you will configure your server1 system to boot into different targets.
Resources:
Machines: server1
Outcomes:
A system booted into different targets.
  • Reset your server1 system.
  1. Configure your server1 system to boot into the multi-user target by default. Do not reboot your server1 system right now, as it will make your graphical console unavailable. 
    1. [root@server1 ~]# systemctl set-default multi-user.target
rm '/etc/systemd/system/default.target'
ln -s '/usr/lib/systemd/system/multi-user.target' '/etc/systemd/system/default.target'
  2. Verify that multi-user is now indeed the new default target. 
    1. [root@server1 ~]# systemctl get-default
multi-user.target
      Alternatively: 
      [root@server1 ~]# ls -l /etc/systemd/system/default.target
lrwxrwxrwx. 1 root root 41 Mar 10 08:00 /etc/systemd/systemd/default.target -> /usr/lib/systemd/system/multi-user.target

Note

In our environment, the grahpical console is not available (except via VNC). If you wanted to set the default target back to the graphical target you would use: 
[root@server1 ~]# systemctl set-default graphical.target
