3.3. Red Hat Network Satellite 5 Configuration Channels
In this section you will learn to:
  • Compare the use of diff/patch with git with RHN Satellite 5
Red Hat Network Satellite 5 has the capability to deploy configuration files to client systems. It does this through a mechanism called configuration channels. Unlike software channels which deploy RPM packages to target systems, configuration channels are used to deploy individual files. The types of configuration files managed by Red Hat Network include text files, binary files, and symbolic links.
Red Hat Network administrators can use command-line tools or the Red Hat Network web interface to create configuration files. The file contents can either be imported or typed in directly. Security attributes of each configuration file are displayed and modified using the web interface. Each file's user and group ownership, permissions, and SELinux context are specified in this way. When command-line tools are used, the security attributes from the original system are used to determine these values.
Red Hat Network Configuration Channels support macro expansion within the contents of text files. There are predefined values that can be referenced, such as host name, IP address, Red Hat Network profile name and ID numbers, as well as custom fields that can be defined in Red Hat Network. This capability allows templates to be created that are adapted when they are deployed to target hosts.
